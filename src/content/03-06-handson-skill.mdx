<De>

**Wählt einen Skill aus eurem Alltag.** Vorschläge falls ihr keine Idee habt:

| Skill | Was er macht |
|---|---|
| `release-notes` | Commits seit letztem Tag → Markdown-Changelog |
| `find-todo` | TODO/FIXME im Repo → strukturierte Liste mit Datei + Zeile |
| `pr-template` | Aus Diff → PR-Beschreibung mit Summary, Changes, Test Plan |
| `dep-audit` | Neue Dependency → Lizenz + Maintainer + Last-Release prüfen |
| `daily-standup` | Eure Commits seit gestern → Standup-Format |

**Schritt 1 (10 min):** SKILL.md skizzieren

<CommandBox lang="bash">
{`mkdir -p .claude/skills/<name>
$EDITOR .claude/skills/<name>/SKILL.md`}
</CommandBox>

Frontmatter mit `name`, `description` (mit 2–3 Beispiel-Triggern), optional `allowed-tools`.

**Schritt 2 (20 min):** Body schreiben — kürzeste sinnvolle Schrittfolge.

**Schritt 3 (10 min):** Testen
1. Neue Session: `claude`
2. Trigger eintippen wie ein User: `mach mir Release-Notes für die letzten 7 Tage`
3. Schaut, ob Skill auto-invoked wird (im Output sichtbar)
4. Wenn nein: `description` schärfen, mehr Trigger-Beispiele

**Schritt 4 (5 min):** Commit + Pull Request

<CommandBox lang="bash">
{`git add .claude/skills/<name>/
git commit -m "feat(skills): add <name> skill"`}
</CommandBox>

</De>

<En>

**Pick a skill from your daily work.** Suggestions if you don't have an idea:

| Skill | What it does |
|---|---|
| `release-notes` | Commits since last tag → markdown changelog |
| `find-todo` | TODO/FIXME in repo → structured list with file + line |
| `pr-template` | From diff → PR description with summary, changes, test plan |
| `dep-audit` | New dependency → check license + maintainer + last release |
| `daily-standup` | Your commits since yesterday → standup format |

**Step 1 (10 min):** Sketch SKILL.md

<CommandBox lang="bash">
{`mkdir -p .claude/skills/<name>
$EDITOR .claude/skills/<name>/SKILL.md`}
</CommandBox>

Frontmatter with `name`, `description` (2–3 example triggers), optionally `allowed-tools`.

**Step 2 (20 min):** Write the body — shortest sensible step list.

**Step 3 (10 min):** Test
1. New session: `claude`
2. Type a trigger like a user: `give me release notes for the last 7 days`
3. Watch if the skill auto-invokes (visible in output)
4. If not: sharpen `description`, add more trigger examples

**Step 4 (5 min):** Commit + pull request

<CommandBox lang="bash">
{`git add .claude/skills/<name>/
git commit -m "feat(skills): add <name> skill"`}
</CommandBox>

</En>
