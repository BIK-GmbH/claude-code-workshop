# <I18n de="Kontext kuratieren" en="Curate Your Context" />

<De>
## Das Kontext-Paradox

> Mehr Kontext ≠ besseres Ergebnis. Zu viel Kontext **verwässert Aufmerksamkeit** und treibt Kosten.

## Vier Kontext-Schichten

| Schicht | Quelle | Wann geladen | Überlebt `/clear`? |
|---|---|---|---|
| **Instructions** | `~/.claude/CLAUDE.md` + `./CLAUDE.md` + `.claude/rules/` | Session-Start, teils lazy bei Pfad-Regeln, nach Compaction | ja |
| **Auto Memory** *(v2.1.59+)* | `~/.claude/projects/<project>/memory/MEMORY.md` | Session-Start (200 Zeilen / 25 KB) | ja |
| **Working Memory** | `Read`-Calls, vorherige Tool-Outputs | im Verlauf der Session | nein |
| **On-Demand** | Subagent-Calls, MCP-Reads, Skill-Loads | bei Bedarf, eigener Context | n/a |

## Faustregeln

- **`/clear` zwischen unverwandten Tasks** — verhindert Context-Pollution. CLAUDE.md + Auto Memory bleiben.
- **`/compact` statt `/clear`** wenn Verlauf wichtig bleibt
- **`/memory` regelmäßig auditieren** — Claude schreibt selbständig Notizen, manchmal seltsame. Mit `/memory` browsen und ggf. löschen
- **Subagents** für Recherche, die Hauptsession nicht braucht
- **Lazy-loaded Architektur-Doku** — Detail in `docs/`, in CLAUDE.md nur Pointer

## Anti-Pattern: „Read everything first"

```
User:  Refactor das gesamte auth-Modul
Claude: [liest 47 Files, 180k Tokens]
        [versucht zu refactoren]
        [bricht ab — Plan zu groß]
```

**Besser:**

```
User:   [Plan Mode] Schau dir auth/ an, schlage 3-Schritt-Refactor-Sequenz vor
Claude: [liest 12 Schlüsseldateien]
        [Plan: Schritt 1 isoliert auth/session, Schritt 2 ...]

User:   Mach Schritt 1
Claude: [liest exakt was es braucht, schreibt, testet]
```

<NoteCard variant="warning">
**Bei zwei fehlgeschlagenen Korrekturen:** `/clear`, neue bessere Prompt schreiben. Weiter im verworrenen Context führt selten zum Ziel.
</NoteCard>

<En>
## The context paradox

> More context ≠ better result. Too much context **dilutes attention** and drives cost.

## Four context layers

| Layer | Source | When loaded | Survives `/clear`? |
|---|---|---|---|
| **Instructions** | `~/.claude/CLAUDE.md` + `./CLAUDE.md` + `.claude/rules/` | Session start, partly lazy for path rules, after compaction | yes |
| **Auto memory** *(v2.1.59+)* | `~/.claude/projects/<project>/memory/MEMORY.md` | Session start (200 lines / 25 KB) | yes |
| **Working memory** | `Read` calls, previous tool outputs | During the session | no |
| **On-demand** | Subagent calls, MCP reads, skill loads | As needed, separate context | n/a |

## Rules of thumb

- **`/clear` between unrelated tasks** — prevents context pollution. CLAUDE.md + auto memory persist.
- **`/compact` instead of `/clear`** if history matters
- **Audit `/memory` periodically** — Claude writes notes autonomously, sometimes odd ones. Browse + prune via `/memory`
- **Subagents** for research the main session doesn't need
- **Lazy-loaded architecture docs** — detail in `docs/`, only pointers in CLAUDE.md

## Anti-pattern: "Read everything first"

```
User:  Refactor the entire auth module
Claude: [reads 47 files, 180k tokens]
        [tries to refactor]
        [bails — plan too big]
```

**Better:**

```
User:   [plan mode] Look at auth/, propose a 3-step refactor sequence
Claude: [reads 12 key files]
        [Plan: step 1 isolates auth/session, step 2 ...]

User:   Do step 1
Claude: [reads exactly what it needs, writes, tests]
```

<NoteCard variant="warning">
**After two failed corrections:** `/clear`, write a better prompt. Continuing in the muddled context rarely succeeds.
</NoteCard>
</En>
