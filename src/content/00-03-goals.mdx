# <I18n de="Was ihr danach könnt" en="What you'll be able to do" />

<De>
Nach dem Workshop seid ihr in der Lage:

1. **Claude Code installieren, konfigurieren und produktiv betreiben** — inklusive Authentifizierung, Modellauswahl und Permissions-Strategie.

2. Eine projektspezifische **`CLAUDE.md`** als „Projektgedächtnis" zu schreiben — Codebasis, Konventionen, Domänen­wissen.

3. **Eigene Skills** zu entwickeln, um wiederkehrende Aufgaben (Code-Generierung, Reviews, Tests) zu standardisieren.

4. **MCP-Server, Slash Commands und Subagents** gezielt einzubinden.

5. Anforderungen so zu strukturieren, dass Claude Code sie zuverlässig in Code überführt — Stichwort **Spec-Driven Development**.

6. KI-generierte Ergebnisse **systematisch zu prüfen, abzusichern und produktiv zu integrieren**.

7. **Die Grenzen des Tools realistisch einzuschätzen** und bewusst zu entscheiden, was delegiert wird — und was nicht.

<NoteCard variant="tip" title="Erfolgs­kriterium">
Ihr verlasst den Workshop mit einem **funktionierenden Setup**, einer **ersten CLAUDE.md** und einem **produktiv ausgelieferten Ergebnis** in eurem eigenen Repository.
</NoteCard>
</De>

<En>
After the workshop you'll be able to:

1. **Install, configure and run Claude Code productively** — including auth, model selection, and a permissions strategy.

2. Write a project-specific **`CLAUDE.md`** as a "project memory" — codebase, conventions, domain knowledge.

3. Build **your own skills** to standardise recurring tasks (code-gen, reviews, tests).

4. Plug in **MCP servers, slash commands and subagents** with intent.

5. Structure requirements so Claude Code reliably turns them into code — **spec-driven development**.

6. **Systematically review, secure and integrate** AI-generated output.

7. **Realistically gauge tool limits** and decide what to delegate — and what not.

<NoteCard variant="tip" title="Success criterion">
You leave with a **working setup**, a **first CLAUDE.md** and a **shipped result** in your own repository.
</NoteCard>
</En>
