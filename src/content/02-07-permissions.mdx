# <I18n de="Permissions" en="Permissions" />

<De>
## Das Permissions-Modell

Claude Code fragt **bei jedem ungewohnten Tool-Call** nach Freigabe. Die wichtigsten Tools:

| Tool | Was passiert | Standard-Verhalten |
|---|---|---|
| **Read** | Datei lesen | meist auto-allow im Repo |
| **Edit / Write** | Datei ändern / erstellen | Bestätigung erforderlich |
| **Bash** | Shell-Command ausführen | Bestätigung pro Command |
| **WebFetch / WebSearch** | Internet | Bestätigung |
| **MCP-Tools** | externe APIs | per Server konfigurierbar |

## Konfiguration

`.claude/settings.json` (per Repo) oder `~/.claude/settings.json` (global):

<CommandBox lang="json">
{`{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff)",
      "Bash(npm test)",
      "Bash(pytest *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)",
      "Bash(curl * | sh)"
    ]
  }
}`}
</CommandBox>

## Modi

| Modus | Was läuft ohne Nachfrage | Wann sinnvoll |
|---|---|---|
| **default** | Lesen | Start, sensible Repos |
| **acceptEdits** | Lesen, Edits, einfache Filesystem-Commands | Iteration an Code, den ihr live reviewt |
| **plan** | Lesen + read-only Shell | Analyse vor Änderungen |
| **auto** | Vieles, mit Safety-Checks | Lange Tasks, weniger Prompt-Fatigue |
| **dontAsk** | Nur vorab erlaubte Tools | CI / locked-down Umgebungen |
| **bypassPermissions** | Fast alles | Nur isolierte Container/VMs |

<CommandBox lang="bash">
claude --permission-mode plan
claude --permission-mode acceptEdits
claude --permission-mode bypassPermissions

# alter Alias für bypassPermissions:
claude --dangerously-skip-permissions
</CommandBox>

<NoteCard variant="warning" title="Sicherheits-Faustregel">
Bypass/YOLO-Mode **niemals** auf produktiven Repos oder mit persönlichen Secrets. Wenn ihr ihn nutzt: in Container/Devcontainer mit klar abgegrenztem Volume.
</NoteCard>

## Empfohlene Team-Settings

- **Lese-Tools** auto-allow im Repo
- **Tests / Linter** auto-allow (geringes Risiko, hohe Frequenz)
- **Git-Commands** auto-allow für `status`, `diff`, `log`; **immer bestätigen** für `push`, `reset --hard`
- **Curl / Wget / sudo** explizit deny oder ask
- **Sensitive Files** explizit deny: `Read(./.env)`, `Read(./.env.*)`, `Read(./secrets/**)`
</De>

<En>
## The permissions model

Claude Code asks for approval on **every unfamiliar tool call**. Key tools:

| Tool | What happens | Default |
|---|---|---|
| **Read** | Read a file | usually auto-allowed in repo |
| **Edit / Write** | Change / create file | confirmation required |
| **Bash** | Execute shell command | per-command confirmation |
| **WebFetch / WebSearch** | Internet | confirmation |
| **MCP tools** | External APIs | configurable per server |

## Configuration

`.claude/settings.json` (per repo) or `~/.claude/settings.json` (global):

<CommandBox lang="json">
{`{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff)",
      "Bash(npm test)",
      "Bash(pytest *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)",
      "Bash(curl * | sh)"
    ]
  }
}`}
</CommandBox>

## Modes

| Mode | What runs without asking | Best for |
|---|---|---|
| **default** | Reads | Starting out, sensitive repos |
| **acceptEdits** | Reads, edits, simple filesystem commands | Iterating on code you review live |
| **plan** | Reads + read-only shell | Analysis before changes |
| **auto** | Many actions, with safety checks | Long tasks, less prompt fatigue |
| **dontAsk** | Only pre-approved tools | CI / locked-down environments |
| **bypassPermissions** | Almost everything | Isolated containers/VMs only |

<CommandBox lang="bash">
claude --permission-mode plan
claude --permission-mode acceptEdits
claude --permission-mode bypassPermissions

# old alias for bypassPermissions:
claude --dangerously-skip-permissions
</CommandBox>

<NoteCard variant="warning" title="Safety rule of thumb">
**Never** use bypass/YOLO mode on production repos or with personal secrets. If you do: container/devcontainer with a clearly bounded volume.
</NoteCard>

## Recommended team settings

- **Read tools** auto-allow in repo
- **Tests / linters** auto-allow (low risk, high frequency)
- **Git commands** auto-allow for `status`, `diff`, `log`; **always confirm** for `push`, `reset --hard`
- **Curl / wget / sudo** explicitly deny or ask
- **Sensitive files** explicitly deny: `Read(./.env)`, `Read(./.env.*)`, `Read(./secrets/**)`
</En>
