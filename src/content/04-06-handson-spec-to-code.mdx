<De>

**Wählt eine Aufgabe aus eurem Backlog**, die ca. 1–2 Stunden Arbeit wäre, wenn ihr es selbst schreibt. Nicht trivial (10 Min), nicht riesig (1 Tag).

**Phase 1 (10 min) — Spec schreiben**

Erstellt `docs/specs/<feature>.md` mit:
- Goal (1 Satz)
- Why (1 Satz)
- Constraints (Bullet-Liste)
- Acceptance Criteria (testbare Aussagen)
- Anti-Goals (was explizit NICHT)
- Files involved (eure Schätzung)

**Phase 2 (10 min) — Plan generieren**

<CommandBox lang="text">
{`> [Plan Mode] Lies docs/specs/<feature>.md. Erstelle einen
  Implementierungsplan mit Schritten, Files, Tests und Risiken.`}
</CommandBox>

Plan iterieren bis er passt. Wenn fertig: in `docs/plans/<feature>.md` ablegen.

**Phase 3 (30 min) — Implementieren**

<CommandBox lang="text">
{`> Mach Schritt 1 aus dem Plan. Schreib den Test zuerst, dann den Code,
  dann lass den Test laufen.

> Mach Schritt 2.

> ...`}
</CommandBox>

Pro Schritt: Test → Code → grün → kurz committen.

**Phase 4 (10 min) — Review**

- `git diff main..HEAD` — alles wie geplant?
- Test-Suite voll laufen lassen
- Manueller Smoke-Check der goldenen Pfade
- Wenn alles grün: PR aufmachen

</De>

<En>

**Pick a backlog item** that would be ~1–2 hours of work if you wrote it yourself. Not trivial (10 min), not huge (1 day).

**Phase 1 (10 min) — Write the spec**

Create `docs/specs/<feature>.md` with:
- Goal (one sentence)
- Why (one sentence)
- Constraints (bullets)
- Acceptance criteria (testable statements)
- Anti-goals (what explicitly NOT)
- Files involved (your estimate)

**Phase 2 (10 min) — Generate plan**

<CommandBox lang="text">
{`> [Plan Mode] Read docs/specs/<feature>.md. Build an implementation
  plan with steps, files, tests and risks.`}
</CommandBox>

Iterate the plan until it fits. When done: drop it into `docs/plans/<feature>.md`.

**Phase 3 (30 min) — Implement**

<CommandBox lang="text">
{`> Do step 1 from the plan. Write the test first, then the code,
  then run the test.

> Do step 2.

> ...`}
</CommandBox>

Per step: test → code → green → small commit.

**Phase 4 (10 min) — Review**

- `git diff main..HEAD` — everything as planned?
- Run the full test suite
- Manual smoke check of golden paths
- All green: open the PR

</En>
