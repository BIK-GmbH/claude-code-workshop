## Beispiel: `release-notes`-Skill

<CommandBox lang="markdown" caption=".claude/skills/release-notes/SKILL.md">
{`---
name: release-notes
description: |
  Erstelle Release Notes aus Commits seit dem letzten Git-Tag.
  Use when the user says "release notes", "changelog", "what changed",
  oder auf Deutsch "release notes erstellen".
allowed-tools:
  - Bash(git log *)
  - Bash(git tag *)
---

# Release Notes

## Vorgehen

1. \`git tag --sort=-creatordate | head -1\` — letzter Tag
2. \`git log <tag>..HEAD --pretty=format:"%h %s"\` — alle Commits seit dem
3. Gruppiere nach Conventional-Commit-Typ:
   - feat → "✨ Features"
   - fix → "🐛 Fixes"
   - chore/docs → "📝 Misc"
   - andere → "Other"
4. Output als Markdown.

## Output-Format

\`\`\`markdown
# Release <next-version>

## ✨ Features
- abc123 — feat: add user profile
- ...

## 🐛 Fixes
- def456 — fix: handle null email
\`\`\``}
</CommandBox>
