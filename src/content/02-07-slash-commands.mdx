# <I18n de="Slash Commands" en="Slash Commands" />

<De>
## Eingebaute Commands (die wichtigsten)

| Command | Was es tut |
|---|---|
| `/help` | Hilfe anzeigen |
| `/clear` | Session-Kontext leeren — Frischstart |
| `/compact` | Verlauf komprimieren statt löschen |
| `/cost` | Kosten dieser Session |
| `/model` | Modell wechseln |
| `/init` | Initiale `CLAUDE.md` für ein Repo generieren |
| `/loop` | Prompt im Intervall wieder feuern *(v2.1.72+, siehe Slide 2.08)* |
| `/memory` | **Auto-Memory** browsen + on/off togglen + CLAUDE.md-Files öffnen *(v2.1.59+)* |
| `/ultrareview` | tiefer Multi-Agent-Code-Review in Cloud-Sandbox |
| `/permissions` | Berechtigungs-Übersicht & -Einstellungen |
| `/skills` | installierte Skills auflisten |
| `/agents` | verfügbare Subagents |
| `/mcp` | MCP-Server-Status |
| `/login` / `/logout` | Auth-Wechsel |

<NoteCard variant="tip" title="`/memory` ist neuer als ihr denkt">
Seit v2.1.59 schreibt Claude **selbständig** Notizen unter `~/.claude/projects/<project>/memory/`. `/memory` öffnet diese Folder zum Inspizieren oder Editieren. Wer aktiv prüfen will was Claude gespeichert hat: einfach `/memory` aufrufen.
</NoteCard>

## Eigene Commands = Skills

Neue Empfehlung: `.claude/skills/<name>/SKILL.md`. Legacy-Dateien unter `.claude/commands/<name>.md` funktionieren weiter und erzeugen denselben `/name`-Aufruf.

<CommandBox lang="markdown">
{`---
name: find-todo
description: Ungelöste TODOs im Code finden und in Issues umwandeln
disable-model-invocation: true
---

Suche im Repo nach TODO/FIXME-Kommentaren.
Pro Treffer: erstelle ein GitHub-Issue mit
- Titel: TODO-Text
- Body: Datei + Zeilennummer + Code-Snippet
- Label: from-todo`}
</CommandBox>

Aufruf: `/skill-name` (z.B. `/find-todo`).

<NoteCard variant="hint">
**2026-Update:** Custom Commands wurden in Skills integriert. Skills sind vorzuziehen, weil sie Helper-Files, Invocation-Control und automatische Relevanz-Erkennung besser unterstützen.
</NoteCard>
</De>

<En>
## Built-in commands (the important ones)

| Command | What it does |
|---|---|
| `/help` | Show help |
| `/clear` | Clear session context — fresh start |
| `/compact` | Compress history instead of clearing |
| `/cost` | Cost of this session |
| `/model` | Switch model |
| `/init` | Generate an initial `CLAUDE.md` for a repo |
| `/loop` | Re-fire a prompt on an interval *(v2.1.72+, see slide 2.08)* |
| `/memory` | Browse **auto memory** + toggle on/off + open CLAUDE.md files *(v2.1.59+)* |
| `/ultrareview` | deep multi-agent code review in a cloud sandbox |
| `/permissions` | Permissions overview & settings |
| `/skills` | List installed skills |
| `/agents` | Available subagents |
| `/mcp` | MCP server status |
| `/login` / `/logout` | Auth switch |

<NoteCard variant="tip" title="`/memory` is newer than you think">
Since v2.1.59 Claude **autonomously** writes notes under `~/.claude/projects/<project>/memory/`. `/memory` opens that folder to inspect or edit. To audit what Claude has saved: just run `/memory`.
</NoteCard>

## Custom commands = skills

New recommendation: `.claude/skills/<name>/SKILL.md`. Legacy files under `.claude/commands/<name>.md` still work and create the same `/name` invocation.

<CommandBox lang="markdown">
{`---
name: find-todo
description: Find unresolved TODOs and turn them into issues
disable-model-invocation: true
---

Scan the repo for TODO/FIXME comments.
For each hit: create a GitHub issue with
- title: TODO text
- body: file + line + code snippet
- label: from-todo`}
</CommandBox>

Invoke: `/skill-name` (e.g. `/find-todo`).

<NoteCard variant="hint">
**2026 update:** Custom commands were merged into skills. Prefer skills because they better support helper files, invocation control, and automatic relevance detection.
</NoteCard>
</En>
