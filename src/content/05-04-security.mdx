<De>
## Die fünf wichtigsten Regeln

### 1. Niemals Secrets in den Kontext

- ❌ API-Keys in `CLAUDE.md`
- ❌ `.env`-Files in `Read`-Calls
- ❌ DB-Connection-Strings in Prompts
- ❌ „remember the prod-DB password is xyz" → landet in **Auto Memory** als Plain-Text
- ✓ Env-Vars + Secrets-Manager + `.gitignore`
- ✓ **`/memory` regelmäßig auditieren** — siehe was Claude selbständig gespeichert hat

### 2. Permissions explizit konfigurieren

`.claude/settings.json`:
- **Allow** häufige sichere Commands (`git status`, `npm test`)
- **Deny** zerstörerische Patterns (`rm -rf`, `git push --force`, `curl | sh`)
- Nichts auto-allow, was Internet-Zugriff oder Production-State berührt

### 3. MCP-Server mit minimalen Privilegien

- DB-MCP mit **Read-Only-Connection-String** (Claude bekommt eigene DB-User)
- GitHub-MCP mit **Token-Scope: nur lesende Repos** für Recherche-Use-Cases
- Slack-MCP nur Channels, in denen Claude posten soll — nicht workspace-weit

### 4. Sandbox für „YOLO"-Workflows

Wenn ihr `--dangerously-skip-permissions` braucht:
- **Devcontainer** mit klar abgegrenztem Volume
- **Keine** Mount des Host-`~/.ssh` oder `~/.aws`
- Network-Policy: ausgehend nur whitelisted Domains

### 5. Audit-Logging

- Bei Enterprise-Setup: Bedrock/Vertex Logging einschalten — alle Prompts + Responses landen in eurem Log-Stream
- DSGVO: Personenbezogene Daten **nicht** an Hosted-LLMs schicken (oder DPA + Region prüfen)

### 6. Auto-Memory-Privacy

- Auto Memory liegt **unverschlüsselt** in `~/.claude/projects/<project>/memory/` als Plain-Markdown
- **Machine-local** — kein Cloud-Sync, kein Versions-Commit. Aber: wer Zugriff auf eure Disk hat, sieht alles
- Inhalt: alles was Claude für „speicherwert" hält — Build-Befehle, Workflow-Habits, manchmal sensible Patterns
- **Disable** via `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` oder `autoMemoryEnabled: false` in Settings
- **Periodisch auditieren:** `/memory` öffnen, durchschauen, irrelevante Notizen löschen
- **Backup-Strategie:** wer Auto Memory zwischen Maschinen sync'en will, braucht eigenen Mechanismus (Stow, Syncthing) — vorsichtig mit Multi-User-Maschinen

<En>
## The five most important rules

### 1. Never put secrets in context

- ❌ API keys in `CLAUDE.md`
- ❌ `.env` files in `Read` calls
- ❌ DB connection strings in prompts
- ❌ "remember the prod DB password is xyz" → lands in **auto memory** as plain text
- ✓ Env vars + secrets manager + `.gitignore`
- ✓ **Audit `/memory` regularly** — see what Claude autonomously saved

### 2. Configure permissions explicitly

`.claude/settings.json`:
- **Allow** common safe commands (`git status`, `npm test`)
- **Deny** destructive patterns (`rm -rf`, `git push --force`, `curl | sh`)
- Nothing auto-allow that touches internet or production state

### 3. MCP servers with minimal privileges

- DB MCP with **read-only connection string** (Claude gets its own DB user)
- GitHub MCP with **token scope: read-only repos** for research use
- Slack MCP only for channels Claude should post to — not workspace-wide

### 4. Sandbox for YOLO workflows

If you need `--dangerously-skip-permissions`:
- **Devcontainer** with a bounded volume
- **No** mount of host `~/.ssh` or `~/.aws`
- Network policy: outbound only to whitelisted domains

### 5. Audit logging

- Enterprise: enable Bedrock/Vertex logging — all prompts + responses in your log stream
- GDPR: do **not** send personal data to hosted LLMs (or check DPA + region)

### 6. Auto-memory privacy

- Auto memory lives **unencrypted** in `~/.claude/projects/<project>/memory/` as plain markdown
- **Machine-local** — no cloud sync, no version-control commit. But: anyone with disk access reads it all
- Content: whatever Claude considers "worth remembering" — build commands, workflow habits, sometimes sensitive patterns
- **Disable** via `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` or `autoMemoryEnabled: false` in settings
- **Audit periodically:** open `/memory`, browse, delete irrelevant notes
- **Backup strategy:** want auto memory synced across machines? roll your own (Stow, Syncthing) — careful on multi-user boxes

<NoteCard variant="warning" title="Real incident">
True story: dev wrote AWS credentials into CLAUDE.md "just to test". CLAUDE.md committed, push, public repo. 8 hours later: $4k bitcoin mining bill. Lesson: secrets do NOT go in files.
</NoteCard>
</En>
