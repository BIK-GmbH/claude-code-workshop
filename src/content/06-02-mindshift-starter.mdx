# <I18n de="Mindshift — der vorbereitete Pfad" en="Mindshift — the pre-baked path" />

<De>
## Was ist Mindshift?

Eine **persönliche AI Knowledge Base** zum Sammeln, Verstehen und langfristigen Behalten von Wissen aus YouTube, Artikeln, PDFs und eigenen Notizen. Inspiriert von [Recall](https://www.recall.it/) — aber als eigenständige Anwendung gebaut, mit pragmatischem MVP und sauberer lokaler Entwicklung.

## Repo clonen

<CommandBox lang="bash">
git clone https://github.com/BIK-GmbH/mindshift-starter.git mindshift
cd mindshift
cat docs/PRD.de.md   # oder PRD.md für englisch
</CommandBox>

Das Repo enthält **nur die PRD** — wir bauen die App im Workshop von Null aus, mit Claude Code.

## Tech-Stack

| Schicht | Technologie |
|---|---|
| **Frontend** | React + Tailwind CSS + Lucide Icons + Vite |
| **Backend** | FastAPI + Python + SQLAlchemy + Alembic + Pydantic Settings |
| **DB** | PostgreSQL (+ pgvector für semantische Suche) |
| **Jobs** | FastAPI Background Tasks → später ARQ/Dramatiq |
| **AI** | OpenAI / Anthropic (über Service-Modul gekapselt) |
| **Deploy** | Railway (Frontend / Backend / Postgres als getrennte Services) |

## MVP-Scope (Abschnitt 8 der PRD)

- **Content Ingestion:** YouTube-URL → Job + Transkript-Fetching
- **Knowledge Cards:** Title, Source, Summaries, Notizen, Tags
- **AI-Zusammenfassungen:** kurz/lang, Key Takeaways, Quiz-Fragen
- **Library UI:** List/Grid, Suche, Filter
- **Notizen pro Card:** simple Markdown-Editor
- **Suche:** Keyword-basiert, optional semantisch via pgvector

**Bewusst NICHT im MVP:** Browser Extension, Mobile App, Team Workspaces, Spaced Repetition Scheduler (kommt in V1.1).

## Empfohlene Workshop-Sprint-Schritte (1–5 von 15)

Aus PRD-Abschnitt 19 — was wir im 75-min-Hands-on schaffen können:

1. **Repo scaffolden** mit `frontend/`, `backend/`, `scripts/`, `docs/` *(via `claude --plan` + lies docs/PRD.de.md)*
2. **FastAPI-App** mit Health-Endpoint und Pydantic-Settings
3. **React-App** mit Tailwind und Lucide
4. **`scripts/start.sh` + `stop.sh`** für lokalen Dev-Workflow
5. **PostgreSQL + erste Migration** für Card/Source/Job-Datenmodell

Schritte 6–15 (Ingestion, AI-Summary, Library-UI, Quiz, Deploy) sind die **Heim-Aufgabe** für die Wochen danach.

<En>
## What is Mindshift?

A **personal AI knowledge base** to capture, understand, and retain knowledge from YouTube, articles, PDFs, and your own notes. Inspired by [Recall](https://www.recall.it/) — built as a standalone app with a pragmatic MVP and clean local development.

## Clone the repo

<CommandBox lang="bash">
git clone https://github.com/BIK-GmbH/mindshift-starter.git mindshift
cd mindshift
cat docs/PRD.md   # or PRD.de.md for German
</CommandBox>

The repo contains **only the PRD** — we build the app from scratch in the workshop with Claude Code.

## Tech stack

| Layer | Technology |
|---|---|
| **Frontend** | React + Tailwind CSS + Lucide Icons + Vite |
| **Backend** | FastAPI + Python + SQLAlchemy + Alembic + Pydantic Settings |
| **DB** | PostgreSQL (+ pgvector for semantic search) |
| **Jobs** | FastAPI Background Tasks → later ARQ/Dramatiq |
| **AI** | OpenAI / Anthropic (wrapped in a service module) |
| **Deploy** | Railway (frontend / backend / postgres as separate services) |

## MVP scope (PRD section 8)

- **Content ingestion:** YouTube URL → job + transcript fetching
- **Knowledge cards:** title, source, summaries, notes, tags
- **AI summaries:** short/long, key takeaways, quiz questions
- **Library UI:** list/grid, search, filter
- **Notes per card:** simple markdown editor
- **Search:** keyword-based, optionally semantic via pgvector

**Deliberately NOT in MVP:** browser extension, mobile app, team workspaces, spaced-repetition scheduler (coming in V1.1).

## Recommended workshop sprint steps (1–5 of 15)

From PRD section 19 — what we can ship in the 75-min hands-on:

1. **Scaffold repo** with `frontend/`, `backend/`, `scripts/`, `docs/` *(via `claude --plan` + read docs/PRD.md)*
2. **FastAPI app** with health endpoint + Pydantic settings
3. **React app** with Tailwind + Lucide
4. **`scripts/start.sh` + `stop.sh`** for local dev workflow
5. **PostgreSQL + first migration** for card/source/job data model

Steps 6–15 (ingestion, AI summary, library UI, quiz, deploy) are **homework** for the weeks after.
