# <I18n de="Wo Claude Code im Markt steht" en="Where Claude Code Sits in the Market" />

<De>
## Die wichtigsten Werkzeuge — Stand Mai 2026

| Tool | Form | Stärke | Typischer Use-Case |
|---|---|---|---|
| **Claude Code** | Terminal-CLI, agentisch | Komplexe Refactorings, große Codebasen, Spec-Driven | Multi-File-Tasks, Architektur-Änderungen |
| **Cursor** | VS-Code-Fork mit AI-Layer | IDE-integriert, Inline-Diffs, Autocomplete | Daily Coding, schnelles Editieren |
| **OpenAI Codex (CLI)** | Terminal + Cloud-Sandbox, async | Fire-and-forget Tasks, lange Hintergrund-Jobs | „Schreib das, ich komme in 20 Min wieder" |
| **GitHub Copilot** | IDE-Plugin, Inline + Chat | Vertraute IDE, Auto-Completion | Mainstream-Inline-Hilfe |
| **OpenCode** | Open-Source-CLI, multi-model | Auto-Commit pro Change, Provider-agnostisch | Terminal-Pair-Programming |
| **Aider** | Open-Source-CLI, git-nativ | Diff-basiert, sehr explizite Edits | Saubere kleine Patches |
| **Continue** | Open-Source IDE-Plugin | Lokale Modelle, anpassbar | Self-hosted Setups |

## Faustregeln

- **Daily IDE-Coding** → Cursor oder Copilot
- **Komplexe Tasks an realer Codebasis** → Claude Code
- **Async „mach das im Hintergrund"** → Codex Cloud
- **Self-hosted / Datenschutz-kritisch** → OpenCode + lokale Modelle, oder Continue

<NoteCard variant="hint">
Viele erfahrene Entwickler nutzen **2 Tools parallel**: ein IDE-Tool (Cursor/Copilot) für daily und ein Terminal-Tool (Claude Code/Codex) für die schweren Sachen.
</NoteCard>

<NoteCard variant="warning">
Diese Folie ist bewusst eine Momentaufnahme. Tool-Positionierung, Pricing, Modell-Defaults und Cloud-Funktionen ändern sich monatlich — vor jedem Workshop kurz gegen offizielle Produktseiten prüfen.
</NoteCard>
</De>

<En>
## The main tools — May 2026 snapshot

| Tool | Form | Strength | Typical use case |
|---|---|---|---|
| **Claude Code** | Terminal CLI, agentic | Complex refactors, large codebases, spec-driven | Multi-file tasks, architecture work |
| **Cursor** | VS-Code fork with AI layer | IDE-integrated, inline diffs, autocomplete | Daily coding, fast editing |
| **OpenAI Codex (CLI)** | Terminal + cloud sandbox, async | Fire-and-forget tasks, long background jobs | "Write this, I'll be back in 20 min" |
| **GitHub Copilot** | IDE plugin, inline + chat | Familiar IDE, autocompletion | Mainstream inline help |
| **OpenCode** | Open-source CLI, multi-model | Auto-commit per change, provider-agnostic | Terminal pair programming |
| **Aider** | Open-source CLI, git-native | Diff-based, very explicit edits | Clean small patches |
| **Continue** | Open-source IDE plugin | Local models, customisable | Self-hosted setups |

## Rules of thumb

- **Daily IDE coding** → Cursor or Copilot
- **Complex tasks on real codebases** → Claude Code
- **Async "do this in the background"** → Codex Cloud
- **Self-hosted / privacy-sensitive** → OpenCode + local models, or Continue

<NoteCard variant="hint">
Many experienced devs run **2 tools in parallel**: one IDE tool (Cursor/Copilot) for daily, one terminal tool (Claude Code/Codex) for the heavy lifts.
</NoteCard>

<NoteCard variant="warning">
This slide is intentionally a snapshot. Tool positioning, pricing, model defaults, and cloud features change monthly — quickly verify against official product pages before each workshop.
</NoteCard>
</En>
