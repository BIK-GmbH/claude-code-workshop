# <I18n de="MCP — Model Context Protocol" en="MCP — Model Context Protocol" />

<De>
## Was ist MCP?

- **Offener Standard von Anthropic** für Tool-Integration in LLMs
- Ein **MCP-Server** stellt Tools/Resources/Prompts bereit
- Claude Code (oder jede MCP-fähige LLM-App) **konsumiert** diese
- Eliminiert Custom-Integrationen pro Tool

## Beispiele

| MCP-Server | Was er bietet |
|---|---|
| **github** | Issues, PRs, Code-Search, Repo-Metadata |
| **postgres** | Schema-Inspektion, Read-Only-Queries |
| **slack** | Channels lesen, Posts schreiben |
| **notion** | Pages lesen, Datenbanken abfragen |
| **filesystem** | Definierter Disk-Bereich (Sandbox) |
| **firecrawl / playwright** | Browser-Automation, Web-Scraping |
| **supabase** | DB + Auth + Storage |

## Konfiguration

Empfohlen: `claude mcp add`. Team-geteilte Server landen in `.mcp.json`, persönliche/local-scoped Server in `~/.claude.json`.

<CommandBox lang="bash">
{`# Remote HTTP ist für Cloud-Tools der bevorzugte Transport
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \\
  --header "Authorization: Bearer $GITHUB_TOKEN"

# Lokaler stdio-Server für direkten Systemzugriff
claude mcp add --transport stdio postgres --scope project -- \\
  npx -y @modelcontextprotocol/server-postgres "$DATABASE_READONLY_URL"`}
</CommandBox>

## Status prüfen

<CommandBox lang="bash">
{`/mcp                       # angeschlossene Server + verfügbare Tools
claude mcp list            # konfigurierte Server
claude mcp get github      # Details zu einem Server

# Debug:
claude --debug "mcp"`}
</CommandBox>

<NoteCard variant="warning">
MCP-Server haben oft **breiten Zugriff** (DB, GitHub-Token, Slack). Projekt-Server aus `.mcp.json` brauchen Approval; für Enterprise `managed-mcp.json` oder Allow-/Denylisten nutzen.
</NoteCard>
</De>

<En>
## What is MCP?

- **Open standard from Anthropic** for tool integration in LLMs
- An **MCP server** exposes tools/resources/prompts
- Claude Code (or any MCP-capable LLM app) **consumes** them
- Eliminates per-tool custom integrations

## Examples

| MCP server | What it offers |
|---|---|
| **github** | Issues, PRs, code search, repo metadata |
| **postgres** | Schema inspection, read-only queries |
| **slack** | Read channels, write posts |
| **notion** | Read pages, query databases |
| **filesystem** | Sandboxed disk area |
| **firecrawl / playwright** | Browser automation, web scraping |
| **supabase** | DB + auth + storage |

## Configuration

Recommended: `claude mcp add`. Team-shared servers go into `.mcp.json`, personal/local-scoped servers into `~/.claude.json`.

<CommandBox lang="bash">
{`# Remote HTTP is preferred for cloud tools
claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \\
  --header "Authorization: Bearer $GITHUB_TOKEN"

# Local stdio server for direct system access
claude mcp add --transport stdio postgres --scope project -- \\
  npx -y @modelcontextprotocol/server-postgres "$DATABASE_READONLY_URL"`}
</CommandBox>

## Inspect status

<CommandBox lang="bash">
{`/mcp                       # connected servers + available tools
claude mcp list            # configured servers
claude mcp get github      # details for one server

# Debug:
claude --debug "mcp"`}
</CommandBox>

<NoteCard variant="warning">
MCP servers often have **broad access** (DB, GitHub token, Slack). Project servers from `.mcp.json` require approval; for enterprise use `managed-mcp.json` or allow/deny lists.
</NoteCard>
</En>
