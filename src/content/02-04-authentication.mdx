# <I18n de="Authentifizierung" en="Authentication" />

<De>
## Auth-Pfade

| Pfad | Wer | Vorteil | Nachteil |
|---|---|---|---|
| **Claude Pro / Max** | Einzel-Devs | Flatrate, kein Cost-Schock | Persönlicher Account, kein Multi-User |
| **Claude Team / Enterprise** | Teams | Zentrale Abrechnung, Admin, Policies | Plan-/Org-Setup nötig |
| **Claude Console / API-Key** | Teams, CI/CD | Pay-per-use, Rollen, SSO möglich | Cost-Monitoring nötig |
| **Bedrock / Vertex** | Enterprise | Compliance, eigene Region, SSO | Setup-Aufwand, langsamer Roll-Out neuer Modelle |
| **Microsoft Foundry** | Azure-nahe Enterprise-Setups | Azure Auth + Governance | Provider-Setup nötig |

## Wechsel zwischen Konten

<CommandBox lang="bash">
/logout                    # in der Session abmelden
claude auth logout         # per CLI abmelden
claude                     # neuer Login-Flow

# Mehrere Provider parallel nutzen — über Env-Vars steuerbar:
ANTHROPIC_API_KEY=sk-ant-... claude
CLAUDE_CODE_USE_BEDROCK=1 claude
CLAUDE_CODE_USE_VERTEX=1 claude
</CommandBox>

Wenn `ANTHROPIC_API_KEY` gesetzt ist, hat er Vorrang vor Subscription-Login. Bei merkwürdigen Auth-Fehlern: `unset ANTHROPIC_API_KEY`, dann `/status` prüfen.

## Token-Tracking

<CommandBox lang="bash">
/cost                      # in laufender Session: was hat dieser Run gekostet?
/usage                     # Übersicht über Session-Verbräuche
/status                    # aktiver Account, Modell, Provider
</CommandBox>

<NoteCard variant="warning">
**Niemals** API-Keys in `CLAUDE.md` oder Repo-Files schreiben — Claude liest die und gibt sie potenziell weiter. Immer über Env-Vars / Secrets-Manager.
</NoteCard>
</De>

<En>
## Auth paths

| Path | Who | Pro | Con |
|---|---|---|---|
| **Claude Pro / Max** | Solo devs | Flat rate, no cost shock | Personal account, no multi-user |
| **Claude Team / Enterprise** | Teams | Central billing, admin, policies | Plan/org setup required |
| **Claude Console / API key** | Teams, CI/CD | Pay-per-use, roles, SSO possible | Cost monitoring required |
| **Bedrock / Vertex** | Enterprise | Compliance, own region, SSO | Setup effort, slower new-model rollout |
| **Microsoft Foundry** | Azure-centric enterprise setups | Azure Auth + governance | Provider setup required |

## Switching between accounts

<CommandBox lang="bash">
/logout                    # log out inside the session
claude auth logout         # log out from the CLI
claude                     # new login flow

# Use multiple providers in parallel — controlled via env vars:
ANTHROPIC_API_KEY=sk-ant-... claude
CLAUDE_CODE_USE_BEDROCK=1 claude
CLAUDE_CODE_USE_VERTEX=1 claude
</CommandBox>

If `ANTHROPIC_API_KEY` is set, it takes precedence over subscription login. For strange auth failures: `unset ANTHROPIC_API_KEY`, then check `/status`.

## Token tracking

<CommandBox lang="bash">
/cost                      # in a running session: what did this run cost?
/usage                     # overview of session usage
/status                    # active account, model, provider
</CommandBox>

<NoteCard variant="warning">
**Never** put API keys in `CLAUDE.md` or repo files — Claude reads those and may leak them. Always via env vars / secrets manager.
</NoteCard>
</En>
