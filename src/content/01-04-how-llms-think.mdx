# <I18n de="Mentalmodell: Wie Claude Code denkt" en="Mental Model: How Claude Code Thinks" />

<De>
## Vier Konzepte, die alles erklären

### 1. Token & Kontextfenster

- LLMs „lesen" und „schreiben" in **Tokens** (≈ ¾ Wort)
- Claude 4.6 Opus hat **bis zu 1 Mio Tokens** Kontext, Sonnet 4.6 200k
- **Alles, was im Kontext liegt, kostet Geld + Zeit + Aufmerksamkeit**
- → „Mehr Kontext rein" ist nicht automatisch besser

### 2. Statelessness pro Turn — und wie Claude Code sie aufhebt

- Auf **Modell-Ebene**: jeder LLM-Call startet bei Null
- Claude Code **schiebt drumherum vier Persistenz-Schichten**, sodass es sich für euch wie ein Gedächtnis anfühlt:

| Schicht | Wer schreibt | Wann geladen |
|---|---|---|
| **System-Prompt + CLAUDE.md** | Du | Jeder Session-Start, voll |
| **Auto Memory** *(NEU ab v2.1.59)* | **Claude selbst** | Jeder Session-Start, erste 200 Zeilen / 25 KB |
| **Working Memory** | Tool-Outputs, `Read`-Calls | Im Verlauf der Session |
| **On-Demand** | Subagents, MCP-Reads, Skills | Bei Bedarf, eigener Context |

→ `/clear` löscht **Working Memory**. CLAUDE.md + Auto Memory bleiben.
→ CLAUDE.md gibt es selbst in **vier Scopes** (Managed / Project / User / Local) — Details in **Modul 2.04**.

### 3. Auto Memory — Claude lernt mit

- Claude entscheidet **selbst** was Speicher­wert hat: Build-Befehle, Debug-Insights, Workflow-Habits
- Speicherort: `~/.claude/projects/<project>/memory/MEMORY.md` + Topic-Files
- **`/memory`** in der Session öffnet, browse, toggle on/off
- „remember that the API tests need Redis" → wird automatisch gespeichert
- Pro git-Repo eigener Memory-Folder, **machine-local** (kein Cloud-Sync, nicht versionsverwaltet)

### 4. Halluzination ist Default

- LLMs **erfinden plausibel klingende Fakten**, wenn sie keine haben
- API-Endpoints, Funktionsnamen, Imports — alles potenziell halluziniert
- Gegenmittel: **Kontext geben** (Doku im Repo, MCP-Server zur DB, klare CLAUDE.md)
- → Verifizieren, nicht vertrauen

<NoteCard variant="warning" title="Anti-Pattern">
„Schreib mir einen Test, der die `userService.getUserById()`-Funktion testet" — wenn diese Funktion nicht im Kontext ist, halluziniert Claude einen plausiblen Stub. Besser: erst `Read userService.ts`, dann fragen.
</NoteCard>

<NoteCard variant="hint" title="Wann wurde das eingeführt">
Auto Memory ist standardmäßig **on ab Claude Code v2.1.59**. `claude --version` prüfen — bei älteren Versionen funktioniert nur Schicht 1 + 3 + 4.
</NoteCard>
</De>

<En>
## Four concepts that explain everything

### 1. Tokens & context window

- LLMs "read" and "write" in **tokens** (≈ ¾ of a word)
- Claude 4.6 Opus has **up to 1M tokens** of context, Sonnet 4.6 200k
- **Everything in the context costs money + time + attention**
- → "More context in" is not automatically better

### 2. Statelessness per turn — and how Claude Code overrides it

- At the **model level**: every LLM call starts from zero
- Claude Code wraps **four persistence layers** around it so it *feels* like memory:

| Layer | Who writes | When loaded |
|---|---|---|
| **System prompt + CLAUDE.md** | You | Every session start, full |
| **Auto Memory** *(NEW since v2.1.59)* | **Claude itself** | Every session start, first 200 lines / 25 KB |
| **Working memory** | Tool outputs, `Read` calls | During the session |
| **On-demand** | Subagents, MCP reads, skills | As needed, separate context |

→ `/clear` wipes **working memory**. CLAUDE.md + auto memory survive.
→ CLAUDE.md itself comes in **four scopes** (Managed / Project / User / Local) — details in **Module 2.04**.

### 3. Auto Memory — Claude learns with you

- Claude decides **itself** what's worth remembering: build commands, debug insights, workflow habits
- Storage: `~/.claude/projects/<project>/memory/MEMORY.md` + topic files
- **`/memory`** in the session opens, browses, toggles on/off
- "remember that the API tests need Redis" → gets saved automatically
- Per git repo own memory folder, **machine-local** (no cloud sync, not version-controlled)

### 4. Hallucination is the default

- LLMs **invent plausible-sounding facts** when they lack them
- API endpoints, function names, imports — all potentially hallucinated
- Antidote: **provide context** (docs in repo, MCP server to DB, clear CLAUDE.md)
- → Verify, don't trust

<NoteCard variant="warning" title="Anti-pattern">
"Write me a test for the `userService.getUserById()` function" — if that function isn't in context, Claude will hallucinate a plausible stub. Better: `Read userService.ts` first, then ask.
</NoteCard>

<NoteCard variant="hint" title="When was this introduced">
Auto memory is on by default **from Claude Code v2.1.59**. Check via `claude --version` — older versions only have layers 1 + 3 + 4.
</NoteCard>
</En>
