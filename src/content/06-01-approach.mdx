# <I18n de="Eigene Projekte" en="Your Projects" />

<De>
## Worum es jetzt geht

Ihr nehmt **eine echte Aufgabe aus eurem Backlog**, die ihr in den nächsten 4 Stunden mit Claude Code umsetzt — von Spec über Plan bis zum Pull Request.

## Auswahlkriterien für die Aufgabe

- **Realistisch:** 1–4 Stunden Aufwand wenn ihr es selbst schreibt
- **Klar abgrenzbar:** kein „und dann auch noch das Authentifizierungs-Refactoring"
- **Nicht zu kritisch:** kein Production-Hotfix, sondern Feature/Refactor mit Test-Sicherung
- **Keine Geheimnisse:** nichts mit unanonymisierten Personendaten oder Production-Secrets

## Ablauf

| Block | Inhalt | Dauer |
|---|---|---|
| 1 | Aufgabe festlegen + Spec schreiben | 30 min |
| 2 | Plan generieren + iterieren | 30 min |
| 3 | Implementieren (Test-First Loop) | 120 min |
| 4 | Code-Review + Pull Request | 30 min |
| 5 | Retro + Q&A im Plenum | 30 min |

<NoteCard variant="tip">
**Pair-Programming empfohlen** — zu zweit arbeiten, einer steuert Claude, einer reviewt live mit. Wechseln nach 30 Min.
</NoteCard>
</De>

<En>
## What this is about

You take **a real task from your backlog** and ship it with Claude Code in the next 4 hours — from spec via plan to pull request.

## Selection criteria for the task

- **Realistic:** 1–4 hours of work if you wrote it yourself
- **Clearly bounded:** no "and also the auth refactor"
- **Not too critical:** no production hotfix, but feature/refactor with test safety net
- **No secrets:** nothing with un-anonymised personal data or production secrets

## Schedule

| Block | Content | Duration |
|---|---|---|
| 1 | Pick task + write spec | 30 min |
| 2 | Generate plan + iterate | 30 min |
| 3 | Implement (test-first loop) | 120 min |
| 4 | Code review + pull request | 30 min |
| 5 | Retro + Q&A in plenary | 30 min |

<NoteCard variant="tip">
**Pair programming recommended** — work in twos, one steers Claude, one reviews live. Swap every 30 min.
</NoteCard>
</En>
