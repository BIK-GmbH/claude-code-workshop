# <I18n de="Hands-on — zwei Pfade" en="Hands-on — two paths" />

<De>
## Worum es jetzt geht

Ihr setzt im 75-min-Block (1-Tag-Format) ein **echtes Stück Software** um — von Spec über Plan bis zu lauffähigem Code. Es gibt **zwei Pfade**, beide gleichwertig:

| Pfad | Wer wählt das | Was ihr braucht |
|---|---|---|
| **A — Eigenes Projekt** | Ihr habt ein Backlog-Item das passt | Lokales Repo, Tests, klare Aufgabe |
| **B — Mindshift-Starter** | Kein eigenes Projekt parat ODER ihr wollt mit dem Trainer gemeinsam pacen | `git clone github.com/BIK-GmbH/mindshift-starter` (siehe nächste Slide) |

Beide Pfade nutzen den gleichen Spec-Driven-Workflow aus Modul 4. Der Unterschied ist nur die Aufgabe.

## Pfad A — eigenes Projekt: Auswahlkriterien

- **Realistisch:** 1–2 Std Aufwand wenn ihr es selbst schreibt
- **Klar abgrenzbar:** kein „und dann auch noch das Authentifizierungs-Refactoring"
- **Nicht zu kritisch:** kein Production-Hotfix, sondern Feature/Refactor mit Test-Sicherung
- **Keine Geheimnisse:** nichts mit unanonymisierten Personendaten oder Production-Secrets

## Pfad B — Mindshift-Starter: warum interessant

- **Vorgegebene PRD** (durchdacht, MVP-scoped) — keine Spec-Schreib-Zeit verloren
- **Trainer pacet vorne** — alle starten gleich, gemeinsame Aha-Momente
- **Realistischer Stack** — React+Tailwind / FastAPI+Postgres / Railway
- **Mitnehmen:** das was ihr im Workshop baut, könnt ihr danach selbst weitertreiben

## Ablauf (gilt für beide Pfade)

| Block | Inhalt | Dauer |
|---|---|---|
| 1 | Spec/PRD lesen + Plan generieren | 15 min |
| 2 | Erste Schritte implementieren (Test-First Loop) | 45 min |
| 3 | Code-Review + Pull Request / Commit | 10 min |
| 4 | Kurz-Retro pro Tisch | 5 min |

<NoteCard variant="tip">
**Pair-Programming empfohlen** — zu zweit arbeiten, einer steuert Claude, einer reviewt live mit. Wechseln nach 20 Min.
</NoteCard>
</De>

<En>
## What this is about

In the 75-min block (1-day format) you ship a **real piece of software** — from spec via plan to working code. **Two paths**, both equivalent:

| Path | Who picks it | What you need |
|---|---|---|
| **A — Your own project** | You have a backlog item that fits | Local repo, tests, clear task |
| **B — Mindshift Starter** | No own project ready OR you want to pace with the trainer | `git clone github.com/BIK-GmbH/mindshift-starter` (see next slide) |

Both paths use the same spec-driven workflow from Module 4. Only the task differs.

## Path A — own project: selection criteria

- **Realistic:** 1–2 hrs of work if you wrote it yourself
- **Clearly bounded:** no "and also the auth refactor"
- **Not too critical:** no production hotfix, but feature/refactor with test safety net
- **No secrets:** nothing with un-anonymised personal data or production secrets

## Path B — Mindshift starter: why interesting

- **Pre-baked PRD** (well-thought-through, MVP-scoped) — no spec-writing time lost
- **Trainer paces from the front** — everyone starts together, shared aha moments
- **Realistic stack** — React+Tailwind / FastAPI+Postgres / Railway
- **Take it home:** what you build in the workshop, you can keep going on after

## Schedule (both paths)

| Block | Content | Duration |
|---|---|---|
| 1 | Read spec/PRD + generate plan | 15 min |
| 2 | Implement first steps (test-first loop) | 45 min |
| 3 | Code review + pull request / commit | 10 min |
| 4 | Quick retro per table | 5 min |

<NoteCard variant="tip">
**Pair programming recommended** — work in twos, one steers Claude, one reviews live. Swap every 20 min.
</NoteCard>
</En>
