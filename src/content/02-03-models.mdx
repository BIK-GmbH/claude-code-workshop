# <I18n de="Welches Modell wofür?" en="Which Model for What?" />

<De>
## Aktuelle Claude-Modelle (Mai 2026)

| Modell | Kontext | Stärke | Cost-Tier | Use-Case |
|---|---|---|---|---|
| **Opus 4.6** | 200k / 1M optional | Komplex, höchste Qualität | $$$$ | Architektur, schwere Refactorings |
| **Opus 4.7** | 200k / 1M optional | Wie 4.6 + besseres Tool-Calling | $$$$ | Lange Agent-Loops |
| **Sonnet 4.6** | 200k / 1M optional | Sehr ausgewogen | $$ | Daily-Driver, 90 % der Tasks |
| **Haiku 4.5** | 200k Tokens | Schnell + günstig | $ | Routine, Linting, einfache Edits |

`opus`, `sonnet`, `haiku` sind **Aliases**. Sie zeigen je nach Provider und Plan auf unterschiedliche konkrete Modellversionen und können sich aktualisieren.

## Wechseln in der Session

<CommandBox lang="bash">
/model                     # interaktiver Picker

# Direkt setzen via Env:
ANTHROPIC_MODEL=claude-sonnet-4-6 claude
ANTHROPIC_MODEL=claude-opus-4-7 claude

# Alias + 1M-Kontext falls Account/Provider es unterstützt:
claude --model opus[1m]
</CommandBox>

## Faustregeln

- **Default für viele Teams/API-Setups:** Sonnet 4.6 — starkes Preis-Leistungs-Verhältnis
- **Komplexe Architektur, mehrere Files, lange Plans:** Opus
- **Routine (Tests schreiben, Boilerplate, Doku-Updates):** Haiku
- **Kostendruck im Team:** mit Sonnet starten, **bei Bedarf** auf Opus eskalieren
- **Plan + Ausführung getrennt:** `opusplan` nutzt Opus im Plan Mode und Sonnet für die Umsetzung

<NoteCard variant="tip">
1M-Kontext ist plan- und providerabhängig. Auf Bedrock/Vertex/Foundry können Aliases hinterherhinken; bei Rollouts konkrete Modell-IDs pinnen.
</NoteCard>
</De>

<En>
## Current Claude models (May 2026)

| Model | Context | Strength | Cost tier | Use case |
|---|---|---|---|---|
| **Opus 4.6** | 200k / optional 1M | Complex, top quality | $$$$ | Architecture, heavy refactors |
| **Opus 4.7** | 200k / optional 1M | Like 4.6 + better tool calls | $$$$ | Long agent loops |
| **Sonnet 4.6** | 200k / optional 1M | Very balanced | $$ | Daily driver, 90% of tasks |
| **Haiku 4.5** | 200k tokens | Fast + cheap | $ | Routine, linting, simple edits |

`opus`, `sonnet`, and `haiku` are **aliases**. They resolve to concrete model versions based on provider and plan, and can update over time.

## Switching mid-session

<CommandBox lang="bash">
/model                     # interactive picker

# Set directly via env:
ANTHROPIC_MODEL=claude-sonnet-4-6 claude
ANTHROPIC_MODEL=claude-opus-4-7 claude

# Alias + 1M context if your account/provider supports it:
claude --model opus[1m]
</CommandBox>

## Rules of thumb

- **Default for many team/API setups:** Sonnet 4.6 — strong price/performance
- **Complex architecture, multi-file, long plans:** Opus
- **Routine (writing tests, boilerplate, doc updates):** Haiku
- **Team cost pressure:** start with Sonnet, **escalate to Opus** as needed
- **Separate planning from execution:** `opusplan` uses Opus in plan mode and Sonnet for implementation

<NoteCard variant="tip">
1M context is plan- and provider-dependent. On Bedrock/Vertex/Foundry aliases may lag; pin concrete model IDs for rollouts.
</NoteCard>
</En>
