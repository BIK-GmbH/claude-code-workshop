# <I18n de="Subagents" en="Subagents" />

<De>
## Was sind Subagents?

- **Eigenständige AI-Worker**, die Claude für spezifische Tasks **parallel** spawnen kann
- Eigenes **Context-Fenster**, eigener **System-Prompt**, eigene **Tool-Whitelist**
- Bis zu **10 parallele Subagents** (Stand 2026)
- Returns ein einzelnes Ergebnis-Statement an die Hauptsession

## Wann nutzen?

| Szenario | Subagent? |
|---|---|
| Recherche zu einem unbekannten Sub-System | **ja** — `code-explorer`-Agent |
| Code-Review eines fertigen Features | **ja** — `code-reviewer`-Agent |
| 5 unabhängige Datei-Edits parallel | **ja** — pro Datei ein Agent |
| Eine schnelle 2-Zeilen-Änderung | nein |
| Linearer Multi-Step-Plan | nein, Plan Mode reicht |

## Definition (`.claude/agents/<name>.md`)

<CommandBox lang="markdown">
{`---
name: code-reviewer
description: |
  Reviews code for bugs, logic errors, security, code quality.
  Use after implementing a feature or before merging.
tools: [Read, Bash(git diff *), Bash(git log *)]
model: opus-4-7
---

You are an expert code reviewer. Read the diff and report:
1. Bugs or logic errors (high confidence only)
2. Security issues
3. Performance regressions
4. Code-quality issues that block readability
5. Missing tests for new behaviour

Format: bulleted list grouped by severity. End with a one-line
verdict: SHIP IT / FIX REQUIRED / NEEDS DISCUSSION.`}
</CommandBox>

## Aufruf

In der Session: **„use the code-reviewer agent to review the diff"** — Claude spawnt den Agent.

<NoteCard variant="tip" title="Killer-Pattern">
**Plan-First + Subagent-Fan-Out:** Hauptsession plant → spawnt 5 Subagents für Recherche → konsolidiert Ergebnisse → schreibt Code. Spart 60–80 % Wall-Clock-Zeit auf großen Tasks.
</NoteCard>
</De>

<En>
## What are subagents?

- **Independent AI workers** Claude can spawn for specific tasks **in parallel**
- Own **context window**, own **system prompt**, own **tool whitelist**
- Up to **10 parallel subagents** (as of 2026)
- Returns a single result statement to the main session

## When to use

| Scenario | Subagent? |
|---|---|
| Research into an unknown sub-system | **yes** — `code-explorer` agent |
| Code review of a finished feature | **yes** — `code-reviewer` agent |
| 5 independent file edits in parallel | **yes** — one agent per file |
| A quick 2-line change | no |
| Linear multi-step plan | no, plan mode is enough |

## Definition (`.claude/agents/<name>.md`)

<CommandBox lang="markdown">

---

You are an expert code reviewer. Read the diff and report:
1. Bugs or logic errors (high confidence only)
2. Security issues
3. Performance regressions
4. Code-quality issues that block readability
5. Missing tests for new behaviour

Format: bulleted list grouped by severity. End with a one-line
verdict: SHIP IT / FIX REQUIRED / NEEDS DISCUSSION.`}
</CommandBox>

## Invocation

In the session: **"use the code-reviewer agent to review the diff"** — Claude spawns the agent.

<NoteCard variant="tip" title="Killer pattern">
**Plan-first + subagent fan-out:** main session plans → spawns 5 subagents for research → consolidates results → writes code. Saves 60–80 % wall-clock time on large tasks.
</NoteCard>
</En>
