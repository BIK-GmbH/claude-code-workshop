# <I18n de="Subagents" en="Subagents" />

<De>
## Was sind Subagents?

- **Eigenständige AI-Worker**, die Claude für spezifische Tasks **parallel** spawnen kann
- Eigenes **Context-Fenster**, eigener **System-Prompt**, eigene **Tool-Whitelist**
- Können im Vordergrund blockieren oder im Hintergrund weiterlaufen
- Rückgabe: komprimiertes Ergebnis an die Hauptsession

## Wann nutzen?

| Szenario | Subagent? |
|---|---|
| Recherche zu einem unbekannten Sub-System | **ja** — `code-explorer`-Agent |
| Code-Review eines fertigen Features | **ja** — `code-reviewer`-Agent |
| 5 unabhängige Recherchen parallel | **ja** — je Subsystem ein Agent |
| 5 unabhängige Datei-Edits parallel | **nur mit klaren Ownership-Grenzen** |
| Eine schnelle 2-Zeilen-Änderung | nein |
| Linearer Multi-Step-Plan | nein, Plan Mode reicht |

## Aufruf

In der Session: **„use the code-reviewer agent to review the diff"** — Claude entscheidet über Delegation. Mit `@agent-code-reviewer` erzwingt ihr den konkreten Agent, mit `claude --agent code-reviewer` startet ihr die ganze Session in diesem Agent-Profil.

<NoteCard variant="tip" title="Killer-Pattern">
**Plan-First + Subagent-Fan-Out:** Hauptsession plant → spawnt mehrere Subagents für unabhängige Recherche → konsolidiert Ergebnisse → schreibt Code. Stark für große Codebasen, solange die Teilaufgaben wirklich unabhängig sind.
</NoteCard>
</De>

<En>
## What are subagents?

- **Independent AI workers** Claude can spawn for specific tasks **in parallel**
- Own **context window**, own **system prompt**, own **tool whitelist**
- Can block in the foreground or continue in the background
- Return a compressed result to the main session

## When to use

| Scenario | Subagent? |
|---|---|
| Research into an unknown sub-system | **yes** — `code-explorer` agent |
| Code review of a finished feature | **yes** — `code-reviewer` agent |
| 5 independent research paths | **yes** — one agent per subsystem |
| 5 independent file edits in parallel | **only with clear ownership boundaries** |
| A quick 2-line change | no |
| Linear multi-step plan | no, plan mode is enough |

## Invocation

In the session: **"use the code-reviewer agent to review the diff"** — Claude decides whether to delegate. Use `@agent-code-reviewer` to force a specific agent, or `claude --agent code-reviewer` to start the whole session in that agent profile.

<NoteCard variant="tip" title="Killer pattern">
**Plan-first + subagent fan-out:** main session plans → spawns several subagents for independent research → consolidates results → writes code. Strong for large codebases when the subtasks are genuinely independent.
</NoteCard>
