<De>
## Was ist CLAUDE.md?

- Markdown-Datei mit **persistenten Anweisungen** für Claude
- Wird beim **Session-Start automatisch als Kontext geladen** und nach Compaction neu eingespielt
- Funktioniert als „Gedächtnis" auf vier verschiedenen Scopes — von **org-wide** bis **persönlich**

## Vier Scopes — wo CLAUDE.md liegen kann

| Scope | Pfad | Geteilt mit | Use-Case |
|---|---|---|---|
| **Managed** *(IT)* | `/Library/Application Support/ClaudeCode/CLAUDE.md` (mac) · `/etc/claude-code/CLAUDE.md` (linux) | alle User der Maschine | Compliance-Regeln, Org-Coding-Standards |
| **Project** | `./CLAUDE.md` oder `./.claude/CLAUDE.md` | das ganze Team via git | Projekt-Architektur, Befehle, Konventionen |
| **User** | `~/.claude/CLAUDE.md` | nur du, **alle Projekte** | Persönliche Präferenzen („antworte auf Deutsch", Editor-Style) |
| **Local** | `./CLAUDE.local.md` (gitignored) | nur du, **dieses Projekt** | Sandbox-URLs, Test-Daten, persönliche Notizen die nicht ins Repo sollen |

**Reihenfolge:** alle vier werden **konkateniert** in den Context. Spezifischere Scopes überschreiben generelle nicht — sie ergänzen sie.

## Project-CLAUDE.md (`./CLAUDE.md`) — die wichtigste

```markdown
# Projektname

## Was ist das?
Ein-Satz-Beschreibung.

## Tech Stack
- Frontend: React 19 + Vite + Tailwind v4
- Backend: FastAPI + Postgres

## Befehle
- Tests: `pytest tests/ -v`
- Build: `npm run build`

## Konventionen
- Conventional Commits (feat/fix/chore/refactor/docs/test)
- Niemals API-Keys committen — nutze .env

## Bekannte Limitierungen
- YouTube-Skills funktionieren nicht auf Railway (Datacenter-IPs)
```

## User-CLAUDE.md (`~/.claude/CLAUDE.md`) — persönlich, alle Projekte

```markdown
# Persönliche Präferenzen

- Antworte immer auf Deutsch
- Code-Beispiele bevorzugt in TypeScript wenn Sprache offen
- Vor jedem Diff: 1-Satz-Zusammenfassung was geändert wird
- Kein "schließlich", "jedoch", "darüber hinaus" — direkter Stil
- Editor: VS Code mit Vim-Bindings (Tabs vs Spaces siehe .editorconfig)
```

## Was NICHT reingehört (in keine der vier)

- ❌ API-Keys, Secrets, Credentials
- ❌ Lange Code-Listings (Claude liest die Files direkt)
- ❌ Ausführliche Doku, die im Repo woanders liegt — verlinken statt einbetten
- ❌ Persönliche Vorlieben in `./CLAUDE.md` — gehört in `~/.claude/CLAUDE.md` (sonst stören sie das Team)

<NoteCard variant="tip" title="Faustregel pro Scope">
- Project = was **das ganze Team** wissen muss
- User = was **dich persönlich** angeht (Sprache, Format-Vorlieben)
- Local = **temporäres Projekt-Wissen** das nicht versionsverwaltet werden soll (Sandbox-URLs, eigene Test-Datasets)
- Managed = **IT-erzwungen**, nicht überschreibbar
</NoteCard>

<NoteCard variant="hint" title="`@path/to/file` für Imports">
CLAUDE.md kann andere Files via `@README.md` einbinden — bis zu 5 Hops tief. Nützlich für `@docs/architecture.md` oder `@~/.claude/my-prefs.md`.
</NoteCard>

<En>
## What is CLAUDE.md?

- Markdown file with **persistent instructions** for Claude
- Loaded into **context automatically** at session start and re-loaded after compaction
- Acts as "memory" at four different scopes — from **org-wide** to **personal**

## Four scopes — where CLAUDE.md can live

| Scope | Path | Shared with | Use case |
|---|---|---|---|
| **Managed** *(IT)* | `/Library/Application Support/ClaudeCode/CLAUDE.md` (mac) · `/etc/claude-code/CLAUDE.md` (linux) | all users on the machine | Compliance, org coding standards |
| **Project** | `./CLAUDE.md` or `./.claude/CLAUDE.md` | the whole team via git | Project architecture, commands, conventions |
| **User** | `~/.claude/CLAUDE.md` | only you, **all projects** | Personal prefs ("answer in English", editor style) |
| **Local** | `./CLAUDE.local.md` (gitignored) | only you, **this project** | Sandbox URLs, test data, personal notes you don't want in git |

**Order:** all four are **concatenated** into context. More specific scopes don't override general ones — they add to them.

## Project CLAUDE.md (`./CLAUDE.md`) — the most important

```markdown
# Project name

## What is it?
One-sentence description.

## Tech stack
- Frontend: React 19 + Vite + Tailwind v4
- Backend: FastAPI + Postgres

## Commands
- Tests: `pytest tests/ -v`
- Build: `npm run build`

## Conventions
- Conventional commits (feat/fix/chore/refactor/docs/test)
- Never commit API keys — use .env

## Known limitations
- YouTube skills don't work on Railway (datacenter IPs)
```

## User CLAUDE.md (`~/.claude/CLAUDE.md`) — personal, all projects

```markdown
# Personal prefs

- Always answer in English
- Code examples in TypeScript by default if language unspecified
- Before any diff: one-sentence summary of what changes
- No "however", "furthermore", "in conclusion" — direct style
- Editor: VS Code with Vim bindings (tabs vs spaces from .editorconfig)
```

## What does NOT go in (any of the four)

- ❌ API keys, secrets, credentials
- ❌ Long code listings (Claude reads files directly)
- ❌ Verbose docs that live elsewhere in the repo — link, don't embed
- ❌ Personal prefs in `./CLAUDE.md` — belongs in `~/.claude/CLAUDE.md` (else it bothers the team)

<NoteCard variant="tip" title="Rule of thumb per scope">
- Project = what the **whole team** needs to know
- User = what's **personal to you** (language, formatting prefs)
- Local = **temporary project knowledge** that shouldn't be version-controlled (sandbox URLs, your test datasets)
- Managed = **IT-enforced**, cannot be overridden
</NoteCard>

<NoteCard variant="hint" title="`@path/to/file` for imports">
CLAUDE.md can include other files via `@README.md` — up to 5 hops deep. Useful for `@docs/architecture.md` or `@~/.claude/my-prefs.md`.
</NoteCard>
