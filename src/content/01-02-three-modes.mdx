# <I18n de="Drei Modi, in denen Teams KI heute begegnen" en="Three modes teams meet AI in today" />

<De>
## 1. Vibe Coding

- KI ad hoc, Prompts kurz, Ergebnisse ohne klare Spezifikation übernommen
- Funktioniert für **Prototypen und Spielereien**
- Skaliert nicht — schwer reviewbar, schwer wartbar
- Symptom: „Es funktioniert, aber niemand weiß warum"

## 2. Tool-Frust

- KI liefert „fast richtigen" Code
- Mehr Review-Aufwand, als sie einspart
- Akzeptanz im Team sinkt; Tools werden wieder abgeschaltet
- Symptom: „Schneller hätte ich es selbst geschrieben"

## 3. Augmented Working ✓

- KI ist Bestandteil eines **spezifikations­getriebenen Workflows**
- Anforderungen, Plan, Code sind nachvollziehbar **getrennt**
- Qualität, Geschwindigkeit, Wartbarkeit steigen messbar
- Symptom: „Wir liefern Dinge, die wir vorher liegen lassen mussten"

<NoteCard variant="tip" title="Kernbotschaft">
Modus 3 ist erlernbar — aber nicht durch Ausprobieren allein. Er erfordert ein **konzeptionelles Verständnis**, wie Claude Code „denkt", wie Kontext gemanagt wird und welche Workflows tragen.
</NoteCard>
</De>

<En>
## 1. Vibe coding

- AI ad hoc, short prompts, results pulled in without a clear spec
- Works for **prototypes and one-offs**
- Doesn't scale — hard to review, hard to maintain
- Symptom: "It works, but nobody knows why"

## 2. Tool frustration

- AI delivers "almost right" code
- More review effort than it saves
- Team adoption drops; tools get switched off again
- Symptom: "I could've written it faster myself"

## 3. Augmented working ✓

- AI is part of a **spec-driven workflow**
- Requirements, plan, code stay **traceably separated**
- Quality, speed, maintainability go up measurably
- Symptom: "We're shipping things we used to defer"

<NoteCard variant="tip" title="Core message">
Mode 3 is learnable — but not by trial and error. It needs a **conceptual understanding** of how Claude Code "thinks", how context is managed, and which workflows pay off.
</NoteCard>
</En>
