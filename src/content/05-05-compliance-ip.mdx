<De>
## Drei Compliance-Themen

### 1. DSGVO / Datenresidenz

- **Wo werden Prompts verarbeitet?** Hängt von Account/Provider/Region ab. Bedrock kann In-Region, Geo oder Global routen — konkret prüfen.
- **Werden Prompts zum Training genutzt?** API/Team/Enterprise: grundsätzlich nein, außer explizitem Opt-in. Free/Pro/Max: abhängig von Privacy-Setting für Model Improvement.
- **Personenbezogene Daten** (Kundennamen, Emails, IDs in Code-Beispielen) — **vorher anonymisieren** oder DPA + EU-Region nutzen.
- **Zero Data Retention** ist Enterprise-/Org-spezifisch und muss aktiv vereinbart/aktiviert sein.

### 2. Code-IP & Lizenzen

- **Wem gehört der KI-generierte Code?** Nicht pauschal beantworten: Vertrag, Jurisdiktion, menschlicher Beitrag und Auftraggeber-Regeln prüfen.
- **Lizenz-Kontamination:** Trainingsdaten können lizenzbehafteten Code enthalten. Enterprise-Verträge können Indemnity/Schutz enthalten; Pro/API/Consumer-Nutzung nicht pauschal.
- **Mitigation:** Lizenz-Scanner (z.B. `licensecheck`, `oss-review-toolkit`) im CI, Snippets > 50 Zeilen manuell prüfen.

### 3. Audit-Trail

- **Was wurde mit KI gemacht?** Wenn Auftraggeber das fragt — könnt ihr antworten?
- **Lösung:** Conventional Commits mit `Co-Authored-By: Claude <noreply@anthropic.com>`-Trailer
- Plus: Spec + Plan in `docs/specs/` und `docs/plans/` committen → vollständige Provenance

## Compliance-Checkliste

- [ ] DPA mit Anthropic / Bedrock / Vertex unterschrieben?
- [ ] EU-Region konfiguriert (falls EU-Daten)?
- [ ] Personenbezogene Daten in Prompts ausgeschlossen / anonymisiert?
- [ ] Lizenz-Scanner im CI aktiv?
- [ ] Audit-Trail in Commits + Specs/Plans?
- [ ] Internes „Was tun bei DSGVO-Anfrage zu KI-Code"-Playbook vorhanden?

<En>
## Three compliance topics

### 1. GDPR / data residency

- **Where are prompts processed?** Depends on account/provider/region. Bedrock can route in-region, geo, or global — verify the concrete setup.
- **Are prompts used for training?** API/Team/Enterprise: generally no, unless explicit opt-in. Free/Pro/Max: depends on the Model Improvement privacy setting.
- **Personal data** (customer names, emails, IDs in code examples) — **anonymise first** or use DPA + EU region.
- **Zero Data Retention** is Enterprise/org-specific and must be contractually enabled.

### 2. Code IP & licences

- **Who owns AI-generated code?** Do not answer generically: check contract, jurisdiction, human contribution, and client rules.
- **Licence contamination:** training data can include licensed code. Enterprise contracts may include indemnity/protection; Pro/API/consumer use is not automatically covered.
- **Mitigation:** licence scanner (e.g. `licensecheck`, `oss-review-toolkit`) in CI, manually review snippets > 50 lines.

### 3. Audit trail

- **What was done with AI?** When the client asks — can you answer?
- **Solution:** conventional commits with `Co-Authored-By: Claude <noreply@anthropic.com>` trailer
- Plus: commit spec + plan in `docs/specs/` and `docs/plans/` → full provenance

## Compliance checklist

- [ ] DPA signed with Anthropic / Bedrock / Vertex?
- [ ] EU region configured (if EU data)?
- [ ] Personal data excluded / anonymised in prompts?
- [ ] Licence scanner in CI active?
- [ ] Audit trail in commits + specs/plans?
- [ ] Internal "what to do on a GDPR request about AI code" playbook?

<NoteCard variant="hint">
**EU AI Act:** the deployment context matters, not just the tool. Developer assistance differs from agentic changes in regulated, safety-critical processes. For medical/finance/HR: involve Legal/Compliance early.
</NoteCard>
</En>
